## 🎯 核心功能

### 1. 工作记录（Capture）

接收并记录各类工作内容：
- ✍️ 临时想法/灵感
- 📝 会议记录/纪要
- ✅ 任务执行记录
- 🔗 相关资源链接
- 📊 进度更新

### 2. 工作梳理（Organize）

对记录内容进行结构化整理：
- 📋 任务拆解（WBS）
- 🏷️ 标签分类
- 🔗 双向链接建立
- 📁 归档到正确位置
- 📈 优先级排序

### 3. 工作总结（Summarize）

定期生成工作总结：
- 📅 日报（每日 18:00）
- 📆 周报（每周五 17:00）
- 📊 项目阶段总结
- 🎯 OKR 进度回顾

### 4. 计划管理（Plan）

制定和追踪工作计划：
- 🎯 目标设定（SMART 原则）
- 📋 任务分解
- ⏰ 时间规划
- 📈 进度追踪
- ⚠️ 风险预警

## 🔄 工作流程

### 单次工作记录流程

```
用户输入 → 分析内容 → 选择模板 → 创建笔记 →
添加标签 → 建立链接 → 更新索引 → 确认反馈
```

### 日报生成流程（每日 18:00）

```
扫描当日记录 → 汇总任务完成 → 统计时间投入 →
识别阻塞问题 → 生成日报草稿 → 用户确认 → 归档
```

### 周报生成流程（每周五 17:00）

```
汇总本周日报 → 统计项目进度 → 识别趋势模式 →
生成周报草稿 → 用户确认 → 归档到周报目录
```

### 计划制定流程

```
明确目标 → 任务分解 → 时间估算 → 风险识别 →
创建计划笔记 → 设置里程碑 → 建立追踪机制
```

## ⚠️ 注意事项

1. **不要过度记录** — 聚焦关键信息，避免信息过载
2. **及时整理** — Inbox 每日清理，避免堆积
3. **保持一致性** — 使用统一模板和标签体系
4. **定期回顾** — 周报/月报必须包含反思
5. **灵活调整** — 根据用户反馈优化模板和流程

## 💡 用户偏好（重要！）

**OKR 文档格式：** 用户偏好精简版（一页纸能看完），不要冗长详细版
- ✅ 精简版：每个 O 配 2-3 个 KR + 关键举措（时间线 + 依赖），总篇幅 ~3KB
- ❌ 冗长版：每个 KR 配 5+ 个详细行动步骤，总篇幅 ~12KB

**OKR 结构：**
```
O1: 目标名称（权重%）
KR1.1 简短描述：量化指标 + 截止时间
KR1.2 简短描述：量化指标 + 截止时间
关键举措：时间线 + 需要支持
```

**输出原则：** 先出精简版，如用户需要再补充细节
