# 内容沉淀与学习消化工作流

当用户给一篇文章、一个链接、一段笔记，要求"学习消化"或"沉淀"时使用。

## 工作流程

### 第一步：分析内容结构

1. 提取标题、来源、核心观点
2. 识别关键特性/功能/建议
3. 找出不足/限制/问题
4. 评估可借鉴点（与现有系统的关联）

### 第二步：关联现有系统

将内容中的想法映射到现有项目/Agent：
- CLM 项目 → `~/Documents/CLM-Obsidian/`
- 工作任务 → `~/Documents/Obsidian-Vault/`
- 个人学习 → `~/Documents/Obsidian-Vault/99-Personal/`
- 资源积累 → `~/Documents/Obsidian-Vault/30-Resources/`

映射规则：
| 内容类型 | 目标目录 | 标签 |
|----------|----------|------|
| 技术文章 | 30-Resources/技术/ | #技术 #学习 |
| 产品功能分析 | 20-Areas/产品分析/ | #产品 #分析 |
| 工作方法论 | 20-Areas/工作方法/ | #方法论 |
| 个人感悟 | 99-Personal/ | #感悟 |
| 会议记录 | 10-Projects/会议/ | #会议 |

### 第三步：生成沉淀文档

模板结构：
```markdown
---
title: "标题"
date: YYYY-MM-DD
source: "来源"
tags: [标签1, 标签2]
status: "processed"
---

# 标题

## 📌 核心内容摘要
(3-5 句话总结)

## 🎯 可借鉴点
(与现有系统的关联，按优先级排序)

## 💡 行动建议
(短期/中期/长期)

## 📝 原文关键信息
(保留原文精华，避免全篇粘贴)

---
*沉淀日期: YYYY-MM-DD | 沉淀人: Agent 名*
*下一步: 具体行动项*
```

### 第四步：存入 Inbox 并通知

1. 写入 `~/Documents/Obsidian-Vault/00-Inbox/YYYY-MM-DD-标题.md`
2. 告知用户已沉淀，并说明：
   - 存到了哪里
   - 提取了几个可借鉴点
   - 建议下一步做什么

## 注意事项

1. **不要全文复制** — 只做摘要和提炼，保留原文链接
2. **关联要具体** — 不要泛泛说"可以用到项目"，要指出具体项目和实现思路
3. **行动项要可执行** — 区分短期（1-2 周）、中期（1 月）、长期（季度）
4. **标签要统一** — 使用已有标签体系，不要每次都发明新标签

## 常见场景

| 场景 | 输入 | 输出 |
|------|------|------|
| 微信公众号文章 | 文章文本 | 分析 + 保存到 Inbox |
| YouTube 视频链接 | URL | web_extract 获取内容 + 分析 |
| 会议记录 | 文本/语音转文字 | 结构化记录 + 标签 |
| 读书笔记 | 读书笔记文本 | 摘要 + 关联现有知识 |
